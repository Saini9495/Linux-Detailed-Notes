# Linux Architecture

Linux follows a layered architecture.

## Architecture Diagram

User
↓
Shell
↓
Kernel
↓
Hardware

---

## 1. Hardware Layer

Physical components:

- CPU
- RAM
- Hard Disk
- Network Card

---

## 2. Kernel Layer

Kernel is the core of Linux.

Responsibilities:

- Process Management
- Memory Management
- Device Management
- File System Management
- Network Management

The kernel directly interacts with hardware.

---

## 3. Shell Layer

Shell acts as an interface between users and the kernel.

Popular Shells:

- Bash
- Zsh
- Fish
- Ksh

Example:

ls
pwd
cd

Commands are sent through the shell.

---

## 4. User Space

Contains:

- Applications
- Utilities
- Services

Examples:

- Chrome
- VS Code
- Docker
- Jenkins

---

## Working Flow

User executes command

↓

Shell interprets command

↓

Kernel processes request

↓

Hardware performs action

↓

Result returned to user

---

## Example

Command:

ls

Process:

User → Shell → Kernel → File System → Output

---

## Summary

Linux architecture consists of:

1. Hardware
2. Kernel
3. Shell
4. User Space

The kernel is the heart of the Linux operating system.
