# SSH Key Authentication

## What is SSH Key Authentication?

SSH Key Authentication allows login without passwords using Public and Private Keys.

---

## Advantages

- More Secure
- Faster Login
- Used in Automation
- Required for CI/CD

---

## Generate SSH Key

```bash
ssh-keygen
```

Output:

```text
id_rsa
id_rsa.pub
```

---

## Key Types

### Private Key

```text
~/.ssh/id_rsa
```

Never share it.

### Public Key

```text
~/.ssh/id_rsa.pub
```

Can be shared.

---

## View Public Key

```bash
cat ~/.ssh/id_rsa.pub
```

---

## Copy Public Key to Server

```bash
ssh-copy-id user@server-ip
```

Example:

```bash
ssh-copy-id ubuntu@13.233.xx.xx
```

---

## Connect Using Key

```bash
ssh user@server-ip
```

or

```bash
ssh -i mykey.pem ubuntu@13.233.xx.xx
```

---

## SSH Agent

Start Agent:

```bash
eval "$(ssh-agent -s)"
```

Add Key:

```bash
ssh-add ~/.ssh/id_rsa
```

---

## Verify Authentication

```bash
ssh -v user@server-ip
```

---

## Best Practices

- Never share private key
- Disable password login
- Rotate keys regularly
- Use passphrase protection
