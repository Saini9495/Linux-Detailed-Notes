# SSH Hardening

## Introduction

SSH Hardening means securing SSH access to reduce the risk of unauthorized access.

---

## SSH Configuration File

```bash
/etc/ssh/sshd_config
```

---

## Change Default Port

Default:

```bash
22
```

Example:

```bash
Port 2222
```

---

## Disable Root Login

```bash
PermitRootLogin no
```

---

## Disable Password Authentication

```bash
PasswordAuthentication no
```

Use SSH Keys instead.

---

## Limit User Access

```bash
AllowUsers ubuntu admin
```

---

## Enable Public Key Authentication

```bash
PubkeyAuthentication yes
```

---

## Disable Empty Passwords

```bash
PermitEmptyPasswords no
```

---

## Restart SSH

```bash
sudo systemctl restart ssh
```

---

## Generate SSH Key

```bash
ssh-keygen -t rsa -b 4096
```

---

## Copy Public Key

```bash
ssh-copy-id user@server-ip
```

---

## SSH Security Checklist

✅ Disable root login

✅ Change default port

✅ Use SSH keys

✅ Enable firewall

✅ Install Fail2Ban

✅ Keep SSH updated

---

## Best Practices

- Use SSH key authentication
- Restrict users
- Monitor login attempts
- Disable unused accounts

---

## Interview Questions

### What is SSH Hardening?

The process of securing SSH access to minimize attack risks.

### Why disable root login?

To reduce the chance of unauthorized administrative access.
