# Linux Security Best Practices

## Introduction

Linux security involves protecting systems, users, applications, and data from unauthorized access and cyber threats.

---

## 1. Keep System Updated

```bash
sudo apt update
sudo apt upgrade -y
```

---

## 2. Use Strong Passwords

Characteristics:

- Minimum 12 characters
- Uppercase letters
- Lowercase letters
- Numbers
- Special characters

---

## 3. Use SSH Keys

Avoid password-based login.

```bash
ssh-keygen
```

---

## 4. Disable Root Login

```bash
PermitRootLogin no
```

---

## 5. Enable Firewall

```bash
sudo ufw enable
```

---

## 6. Install Fail2Ban

Protect against brute-force attacks.

```bash
sudo apt install fail2ban
```

---

## 7. Remove Unused Packages

```bash
sudo apt autoremove
```

---

## 8. Monitor Logs

System Logs:

```bash
/var/log/syslog
```

Authentication Logs:

```bash
/var/log/auth.log
```

---

## 9. Principle of Least Privilege

Give users only required permissions.

---

## 10. Backup Important Data

Example:

```bash
rsync -av source destination
```

---

## 11. Encrypt Sensitive Data

Tools:

- GPG
- OpenSSL
- LUKS

---

## 12. Regular Security Audits

Commands:

```bash
last
who
w
netstat
ss
```

---

## Linux Security Layers

Physical Security
↓
User Security
↓
Network Security
↓
Application Security
↓
Data Security

---

## Security Checklist

✅ Update system

✅ Configure firewall

✅ Install Fail2Ban

✅ Secure SSH

✅ Monitor logs

✅ Backup data

✅ Use least privilege

✅ Remove unused services

## Interview Questions

### What is the Principle of Least Privilege?

Users should receive only the permissions necessary to perform their tasks.

### Why are Linux updates important?

Updates fix security vulnerabilities and bugs.

### What are the most important Linux security tools?

- UFW
- Fail2Ban
- SSH
- SELinux
- AppArmor
