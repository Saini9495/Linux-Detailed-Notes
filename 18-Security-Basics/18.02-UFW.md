# UFW (Uncomplicated Firewall)

## Introduction

UFW is a simple frontend for managing iptables firewall rules.

Ubuntu uses UFW by default.

---

## Install UFW

```bash
sudo apt update
sudo apt install ufw
```

---

## Check Status

```bash
sudo ufw status
```

Output:

```bash
Status: inactive
```

---

## Enable UFW

```bash
sudo ufw enable
```

---

## Disable UFW

```bash
sudo ufw disable
```

---

## Allow Services

### Allow SSH

```bash
sudo ufw allow ssh
```

### Allow HTTP

```bash
sudo ufw allow 80
```

### Allow HTTPS

```bash
sudo ufw allow 443
```

---

## Deny Ports

```bash
sudo ufw deny 23
```

---

## Delete Rule

```bash
sudo ufw delete allow 80
```

---

## View Numbered Rules

```bash
sudo ufw status numbered
```

---

## Reload UFW

```bash
sudo ufw reload
```

---

## Check Logs

```bash
sudo less /var/log/ufw.log
```

---

## Useful Commands

```bash
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
```

---

## Best Practices

- Keep SSH open before enabling firewall
- Allow only required ports
- Regularly review rules
- Remove unused rules

---

## Interview Questions

### What is UFW?

A user-friendly firewall management tool for Linux.

### Why use UFW instead of iptables?

Because UFW simplifies firewall configuration.
