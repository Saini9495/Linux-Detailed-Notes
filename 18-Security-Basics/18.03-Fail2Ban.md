# Fail2Ban

## Introduction

Fail2Ban is an intrusion prevention tool that protects Linux servers from brute-force attacks.

It scans log files and blocks suspicious IP addresses.

---

## How Fail2Ban Works

Attacker
↓
Multiple Failed Login Attempts
↓
Fail2Ban Detects
↓
Firewall Blocks IP

---

## Install Fail2Ban

Ubuntu:

```bash
sudo apt update
sudo apt install fail2ban
```

---

## Check Status

```bash
sudo systemctl status fail2ban
```

---

## Enable Service

```bash
sudo systemctl enable fail2ban
```

---

## Start Service

```bash
sudo systemctl start fail2ban
```

---

## Configuration File

```bash
/etc/fail2ban/jail.local
```

Example:

```ini
[sshd]

enabled = true
port = ssh
maxretry = 5
bantime = 600
findtime = 600
```

---

## Check Active Jails

```bash
sudo fail2ban-client status
```

---

## Check SSH Jail

```bash
sudo fail2ban-client status sshd
```

---

## Unban IP

```bash
sudo fail2ban-client set sshd unbanip IP_ADDRESS
```

---

## Logs

```bash
sudo tail -f /var/log/fail2ban.log
```

---

## Advantages

- Prevents brute-force attacks
- Automatic IP banning
- Works with SSH
- Lightweight

---

## Best Practices

- Protect SSH service
- Set proper ban time
- Monitor logs regularly
- Use strong passwords
