# Linux Backup Commands

## Introduction

Backups help recover data in case of accidental deletion, hardware failure, or cyber attacks.

## Common Backup Commands

### tar

```bash
tar -cvf backup.tar myfolder
```

### tar + gzip

```bash
tar -czvf backup.tar.gz myfolder
```

### rsync

```bash
rsync -av source/ destination/
```

### cp

```bash
cp -r project backup_project
```

## Backup Types

### Full Backup

Copies all data.

### Incremental Backup

Copies only changed files.

### Differential Backup

Copies changes since last full backup.

## Real World Examples

### Backup Website Files

```bash
tar -czvf website_backup.tar.gz /var/www/html
```

### Backup User Home

```bash
tar -czvf home_backup.tar.gz /home/ubuntu
```

### Backup Database Dump

```bash
mysqldump -u root -p database > db_backup.sql
```

### Sync Files to Another Server

```bash
rsync -av /home/ubuntu user@server:/backup
```

## Automated Backup using Cron

### Edit Crontab

```bash
crontab -e
```

### Daily Backup at Midnight

```bash
0 0 * * * tar -czvf /backup/home.tar.gz /home/ubuntu
```

## Best Practices

- Store backups in multiple locations
- Test backups regularly
- Automate backups
- Compress backup files
- Use versioning

## Mini Project

### Daily Backup Script

```bash
#!/bin/bash

DATE=$(date +%F)

tar -czvf backup-$DATE.tar.gz /home/ubuntu

echo "Backup Completed"
```

Run:

```bash
chmod +x backup.sh

./backup.sh
```

## Summary

- Backup is critical for disaster recovery
