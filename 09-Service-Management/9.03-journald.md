# Journald

## Introduction

Journald is the logging service provided by systemd.

It collects:

- System logs
- Kernel logs
- Service logs
- Application logs

## View Logs

```bash
journalctl
```

## Recent Logs

```bash
journalctl -n 50
```

## Follow Logs in Real Time

```bash
journalctl -f
```

## Service Specific Logs

```bash
journalctl -u nginx
```

## Logs Since Boot

```bash
journalctl -b
```

## Logs for Today

```bash
journalctl --since today
```

## Logs for Last Hour

```bash
journalctl --since "1 hour ago"
```

## Real Example

```bash
journalctl -u docker
```

Displays Docker service logs.

## Benefits

- Centralized logging
- Easy troubleshooting
- Service-specific logs
- Persistent logs

## Summary

Journald is the default logging system used by systemd-based Linux distributions.
