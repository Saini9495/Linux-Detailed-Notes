# Log Management in Linux

## What are Logs?

Logs are records of events occurring on a Linux system.

They help administrators:

- Troubleshoot issues
- Monitor services
- Analyze system behavior
- Detect security threats

---

## Log Directory

Most logs are stored in:

```bash
/var/log
```

View logs:

```bash
cd /var/log
ls
```

---

## Important Log Files

### System Logs

```bash
/var/log/syslog
```

### Authentication Logs

```bash
/var/log/auth.log
```

### Kernel Logs

```bash
/var/log/kern.log
```

### Boot Logs

```bash
/var/log/boot.log
```

---

## Useful Commands

### View Entire Log

```bash
cat /var/log/syslog
```

### View Last 20 Lines

```bash
tail -20 /var/log/syslog
```

### Monitor Logs Live

```bash
tail -f /var/log/syslog
```

### Search Logs

```bash
grep "error" /var/log/syslog
```

---

## Log Rotation

Linux uses logrotate to manage log files.

Configuration:

```bash
/etc/logrotate.conf
```

Check configuration:

```bash
cat /etc/logrotate.conf
```

---

## Real World Example

Monitor SSH login attempts:

```bash
tail -f /var/log/auth.log
```

---

## Best Practices

- Regularly monitor logs
- Rotate logs to save disk space
- Secure sensitive log files
- Archive old logs

---

## Summary

Log management is essential for:

- Troubleshooting
- Security monitoring
- System administration
- Performance analysis
