# Special Permissions in Linux

Linux provides three special permissions:

1. SUID
2. SGID
3. Sticky Bit

---

## SUID (Set User ID)

Allows a user to execute a file with the owner's permissions.

### Example

```bash
chmod u+s file
```

Check:

```bash
ls -l
```

Output:

```bash
-rwsr-xr-x
```

### Numeric Value

```text
4
```

Example:

```bash
chmod 4755 file
```

---

## SGID (Set Group ID)

Runs a file with group permissions.

For directories, newly created files inherit the directory group.

### Example

```bash
chmod g+s folder
```

Output:

```bash
drwxr-sr-x
```

### Numeric Value

```text
2
```

Example:

```bash
chmod 2755 folder
```

---

## Sticky Bit

Prevents users from deleting files owned by others.

Mostly used on shared directories like /tmp.

### Example

```bash
chmod +t folder
```

Output:

```bash
drwxrwxrwt
```

### Numeric Value

```text
1
```

Example:

```bash
chmod 1755 folder
```

---

## Combined Special Permissions

| Permission | Value |
|------------|--------|
| SUID | 4 |
| SGID | 2 |
| Sticky Bit | 1 |

Example:

```bash
chmod 7755 file
```

---

## Check Special Permissions

```bash
ls -l
```

Look for:

```text
s = SUID / SGID
t = Sticky Bit
```

## Real-World Use Cases

- passwd command uses SUID
- Shared team directories use SGID
- /tmp directory uses Sticky Bit
