# RAID (Redundant Array of Independent Disks)

## What is RAID?

RAID combines multiple disks to improve:

- Performance
- Redundancy
- Fault Tolerance

## RAID Levels

### RAID 0 (Striping)

Minimum Disks: 2

Advantages:
- High Performance

Disadvantages:
- No Fault Tolerance

Diagram:

Disk1: A B
Disk2: C D

### RAID 1 (Mirroring)

Minimum Disks: 2

Advantages:
- Data Protection

Disadvantages:
- 50% Storage Loss

Diagram:

Disk1: A B C
Disk2: A B C

### RAID 5

Minimum Disks: 3

Advantages:
- Performance + Redundancy

Disadvantages:
- One Disk Failure Allowed

### RAID 6

Minimum Disks: 4

Advantages:
- Two Disk Failures Allowed

### RAID 10

Minimum Disks: 4

Combination of:

RAID 1 + RAID 0

Best balance of:

- Speed
- Fault Tolerance

## Create RAID Using mdadm

Install:

```bash
sudo apt install mdadm
```

Create RAID 1:

```bash
mdadm --create --verbose /dev/md0 \
--level=1 \
--raid-devices=2 \
/dev/sdb /dev/sdc
```

Check RAID:

```bash
cat /proc/mdstat
```

## RAID Management

Show Details:

```bash
mdadm --detail /dev/md0
```

Stop RAID:

```bash
mdadm --stop /dev/md0
```

## RAID Comparison

| RAID | Min Disks | Fault Tolerance |
|--------|-----------|----------------|
| RAID 0 | 2 | No |
| RAID 1 | 2 | Yes |
| RAID 5 | 3 | 1 Disk |
| RAID 6 | 4 | 2 Disks |
| RAID 10 | 4 | Multiple |
