# Logical Volume Manager (LVM)

## Introduction

LVM (Logical Volume Manager) is a storage management system in Linux that provides a flexible way to manage disk storage.

Traditional partitioning creates fixed-size partitions. If a partition becomes full, resizing can be difficult.

LVM solves this problem by allowing storage to be resized dynamically.

---

# Why Use LVM?

## Advantages

- Easy disk resizing
- Online storage expansion
- Better disk management
- Snapshot support
- Multiple disks can be combined
- Flexible storage allocation

---

# LVM Architecture

LVM consists of three main layers:

1. Physical Volume (PV)
2. Volume Group (VG)
3. Logical Volume (LV)

```
Disk → PV → VG → LV → Filesystem
```

---

# Physical Volume (PV)

A Physical Volume is an actual storage device.

Examples:

- /dev/sdb
- /dev/sdc
- /dev/nvme0n1

Create Physical Volume:

```bash
sudo pvcreate /dev/sdb
```

Display PV Information:

```bash
sudo pvdisplay
```

List PVs:

```bash
sudo pvs
```

---

# Volume Group (VG)

A Volume Group combines one or more physical volumes.

Example:

```bash
sudo vgcreate my_vg /dev/sdb
```

Display VG:

```bash
sudo vgdisplay
```

List VGs:

```bash
sudo vgs
```

---

# Logical Volume (LV)

Logical Volumes act like normal partitions.

Create LV:

```bash
sudo lvcreate -L 5G -n my_lv my_vg
```

Where:

- L = Size
- n = Logical Volume Name

Display LV:

```bash
sudo lvdisplay
```

List LV:

```bash
sudo lvs
```

# Creating a Complete LVM Setup

## Step 1: Check Available Disks

```bash
lsblk
```

---

## Step 2: Create Physical Volume

```bash
sudo pvcreate /dev/sdb
```

---

## Step 3: Create Volume Group

```bash
sudo vgcreate my_vg /dev/sdb
```

---

## Step 4: Create Logical Volume

```bash
sudo lvcreate -L 10G -n my_lv my_vg
```

---

## Step 5: Create Filesystem

```bash
sudo mkfs.ext4 /dev/my_vg/my_lv
```

---

## Step 6: Create Mount Point

```bash
sudo mkdir /data
```

---

## Step 7: Mount Logical Volume

```bash
sudo mount /dev/my_vg/my_lv /data
```

---

## Step 8: Verify

```bash
df -h
```

---

# Extending a Logical Volume

Check free space:

```bash
sudo vgs
```

Extend LV:

```bash
sudo lvextend -L +5G /dev/my_vg/my_lv
```

Resize filesystem:

```bash
sudo resize2fs /dev/my_vg/my_lv
```

Verify:

```bash
df -h
```

---

# Reducing a Logical Volume

⚠ Always take backup before reducing.

Unmount:

```bash
sudo umount /data
```

Check filesystem:

```bash
sudo e2fsck -f /dev/my_vg/my_lv
```

Reduce filesystem:

```bash
sudo resize2fs /dev/my_vg/my_lv 5G
```

Reduce LV:

```bash
sudo lvreduce -L 5G /dev/my_vg/my_lv
```

---

# Extending a Volume Group

Add a new disk:

```bash
sudo pvcreate /dev/sdc
```

Add to existing VG:

```bash
sudo vgextend my_vg /dev/sdc
```

Verify:

```bash
sudo vgdisplay
```

---

# Removing LVM

Unmount LV:

```bash
sudo umount /data
```

Remove LV:

```bash
sudo lvremove /dev/my_vg/my_lv
```

Remove VG:

```bash
sudo vgremove my_vg
```

Remove PV:

```bash
sudo pvremove /dev/sdb
```

---

# LVM Snapshots

Snapshots create a point-in-time copy of a logical volume.

Create Snapshot:

```bash
sudo lvcreate -L 2G -s -n snapshot1 /dev/my_vg/my_lv
```

View Snapshot:

```bash
sudo lvs
```

Benefits:

- Backup
- Recovery
- Testing

# Interview Questions

## What is LVM?

LVM is a storage management solution that allows flexible disk management using Physical Volumes, Volume Groups, and Logical Volumes.

---

## What are the components of LVM?

- Physical Volume (PV)
- Volume Group (VG)
- Logical Volume (LV)

---

## What is a Volume Group?

A storage pool created using one or more Physical Volumes.

---

## What is a Snapshot?

A point-in-time copy of a logical volume used for backup and recovery.
