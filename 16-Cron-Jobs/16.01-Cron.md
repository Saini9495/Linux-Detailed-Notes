# ⏰ Cron in Linux

## What is Cron?

Cron is a time-based job scheduler in Linux and Unix systems.

It allows users to schedule commands or scripts to run automatically at specific times and dates.

Cron runs in the background as a daemon called:

```bash
crond
```

or

```bash
cron
```

---

## Why Use Cron?

Cron is used to automate repetitive tasks such as:

- Database Backups
- Log Cleanup
- System Monitoring
- Sending Emails
- Running Scripts
- Restarting Services

---

## Real-Life Example

Imagine you need to take a database backup every day at midnight.

Instead of manually running the backup command daily, Cron can automate it.

---

## How Cron Works

Cron daemon continuously checks:

```bash
/etc/crontab
```

and user cron jobs.

When the scheduled time matches, Cron executes the command automatically.

---

## Check Cron Service Status

Ubuntu:

```bash
sudo systemctl status cron
```

CentOS/RHEL:

```bash
sudo systemctl status crond
```

---

## Start Cron Service

```bash
sudo systemctl start cron
```

Enable at Boot:

```bash
sudo systemctl enable cron
```

---

## Restart Cron Service

```bash
sudo systemctl restart cron
```

---

## Verify Cron Service

```bash
ps -ef | grep cron
```

Output:

```bash
root 1234 ? 00:00:00 cron
```

---

## Cron Job Components

Cron jobs consist of:

```text
Minute Hour Day Month DayOfWeek Command
```

Example:

```bash
0 9 * * * /home/ubuntu/script.sh
```

Meaning:

- Minute → 0
- Hour → 9 AM
- Every Day
- Every Month
- Every Weekday
- Run script.sh
