# 📅 Crontab in Linux

## What is Crontab?

Crontab stands for:

```text
CRON TABLE
```

It is a configuration file that stores scheduled cron jobs.

Each user can have their own crontab file.

---

## View Existing Cron Jobs

```bash
crontab -l
```

Example:

```bash
0 2 * * * /home/ubuntu/backup.sh
```

---

## Edit Cron Jobs

```bash
crontab -e
```

This opens the crontab editor.

---

## Remove All Cron Jobs

```bash
crontab -r
```

Warning:

This permanently removes all cron jobs.

---

## Cron Job Syntax

```text
* * * * * command
│ │ │ │ │
│ │ │ │ └── Day of Week (0-7)
│ │ │ └──── Month (1-12)
│ │ └────── Day of Month (1-31)
│ └──────── Hour (0-23)
└────────── Minute (0-59)
```

---

## Special Characters

### Asterisk (*)

Means every value.

```bash
* * * * *
```

Runs every minute.

---

### Comma (,)

Multiple values.

```bash
0 8,12,18 * * *
```

Runs at 8AM, 12PM and 6PM.

---

### Hyphen (-)

Range.

```bash
0 9-17 * * *
```

Runs every hour between 9AM and 5PM.

---

### Slash (/)

Interval.

```bash
*/5 * * * *
```

Runs every 5 minutes.

---

## Common Examples

### Every Minute

```bash
* * * * * command
```

---

### Every 5 Minutes

```bash
*/5 * * * * command
```

---

### Every Hour

```bash
0 * * * * command
```

---

### Every Day at Midnight

```bash
0 0 * * * command
```

---

### Every Sunday

```bash
0 0 * * 0 command
```

---

### Every Monday at 9 AM

```bash
0 9 * * 1 command
```

---

## System-wide Crontab

Location:

```bash
/etc/crontab
```

View:

```bash
cat /etc/crontab
```

---

## User Crontab Location

```bash
/var/spool/cron/crontabs/
```

---

## Summary

Crontab is used to create, edit, view and manage scheduled cron jobs.
