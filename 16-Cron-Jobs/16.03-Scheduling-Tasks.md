# 🚀 Scheduling Tasks with Cron

## Introduction

Cron is mainly used to automate tasks.

In DevOps and System Administration, task scheduling is extremely important.

---

# 1. Backup Scheduling

Create script:

```bash
nano backup.sh
```

Script:

```bash
#!/bin/bash

tar -czf /backup/home_backup.tar.gz /home
```

Make executable:

```bash
chmod +x backup.sh
```

Schedule:

```bash
0 1 * * * /home/ubuntu/backup.sh
```

Runs every day at 1 AM.

---

# 2. Log Cleanup Automation

Delete logs older than 7 days.

Command:

```bash
find /var/log -type f -mtime +7 -delete
```

Cron Entry:

```bash
0 2 * * * find /var/log -type f -mtime +7 -delete
```

Runs daily at 2 AM.

---

# 3. Disk Usage Monitoring

Script:

```bash
df -h > /home/ubuntu/disk-report.txt
```

Cron:

```bash
0 */6 * * * df -h > /home/ubuntu/disk-report.txt
```

Runs every 6 hours.

---

# 4. Service Monitoring

Check nginx status.

Script:

```bash
systemctl is-active nginx
```

Cron:

```bash
*/10 * * * * systemctl is-active nginx
```

Runs every 10 minutes.

---

# 5. Database Backup

MySQL Backup:

```bash
mysqldump -u root -p database_name > backup.sql
```

Cron:

```bash
0 0 * * * mysqldump -u root -p database_name > backup.sql
```

Runs every midnight.

---

# 6. Auto Reboot Server

Weekly Reboot:

```bash
0 3 * * 0 reboot
```

Runs every Sunday at 3 AM.

---

# 7. Send Scheduled Report

Example:

```bash
mail -s "Daily Report" admin@example.com < report.txt
```

Cron:

```bash
0 8 * * * mail -s "Daily Report" admin@example.com < report.txt
```

Runs daily at 8 AM.

---

# Useful Cron Schedules

| Schedule | Meaning |
|-----------|----------|
| * * * * * | Every minute |
| */5 * * * * | Every 5 minutes |
| */10 * * * * | Every 10 minutes |
| 0 * * * * | Every hour |
| 0 0 * * * | Daily midnight |
| 0 6 * * * | Daily at 6 AM |
| 0 12 * * 1 | Every Monday |
| 0 0 1 * * | First day of month |
| 0 0 1 1 * | Every January 1 |

# Cron Logs

Ubuntu:

```bash
grep CRON /var/log/syslog
```

CentOS:

```bash
grep CRON /var/log/cron
```

---

# Best Practices

- Use absolute paths.
- Test scripts manually before scheduling.
- Redirect output to log files.
- Monitor cron execution regularly.
- Keep backup jobs documented.

Example:

```bash
0 1 * * * /home/ubuntu/backup.sh >> backup.log 2>&1
```
