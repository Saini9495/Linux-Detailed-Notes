# journalctl Command

## Introduction

journalctl is used to view logs collected by systemd-journald.

It provides centralized logging for:

- Kernel
- Services
- Applications
- System Events

---

# Basic Syntax

```bash
journalctl
```

Displays all logs.

---

# View Latest Logs

```bash
journalctl -n 20
```

Shows last 20 entries.

---

# Follow Logs Live

```bash
journalctl -f
```

Similar to:

```bash
tail -f
```

---

# View Logs for Specific Service

Example:

```bash
journalctl -u ssh
```

For nginx:

```bash
journalctl -u nginx
```

---

# View Today's Logs

```bash
journalctl --since today
```

---

# View Logs from Last Hour

```bash
journalctl --since "1 hour ago"
```

---

# View Logs by Priority

Errors only:

```bash
journalctl -p err
```

Warnings:

```bash
journalctl -p warning
```

Critical:

```bash
journalctl -p crit
```

---

# View Boot Logs

Current Boot:

```bash
journalctl -b
```

Previous Boot:

```bash
journalctl -b -1
```

---

# View Kernel Logs

```bash
journalctl -k
```

---

# Disk Usage of Journal Logs

```bash
journalctl --disk-usage
```

---

# Clear Old Logs

Keep last 7 days:

```bash
journalctl --vacuum-time=7d
```

Keep only 500 MB:

```bash
journalctl --vacuum-size=500M
```

---

# Advantages of journalctl

- Centralized Logging
- Easy Searching
- Persistent Logs
- Service-specific Logs
- Structured Output

---
