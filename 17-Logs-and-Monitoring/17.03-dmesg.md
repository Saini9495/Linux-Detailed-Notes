# dmesg Command

## Introduction

dmesg displays messages from the Linux kernel ring buffer.

Used for:

- Hardware Detection
- Driver Issues
- Boot Messages
- USB Devices
- Disk Errors

---

# Basic Syntax

```bash
dmesg
```

Displays kernel messages.

---

# View Latest Messages

```bash
dmesg | tail
```

---

# Human Readable Timestamps

```bash
dmesg -T
```

Example:

```bash
[Mon Jun 15 10:20:00 2026]
```

---

# Search USB Devices

```bash
dmesg | grep USB
```

---

# Search Hard Disk Information

```bash
dmesg | grep sda
```

---

# Search Errors

```bash
dmesg | grep error
```

---

# Monitor New Kernel Messages

```bash
dmesg -w
```

Real-time monitoring.

---

# Common Use Cases

## USB Not Detected

```bash
dmesg | grep usb
```

## Network Card Issues

```bash
dmesg | grep eth
```

## Driver Problems

```bash
dmesg | grep driver
```

## Hardware Failures

```bash
dmesg | grep fail
```

---

# Difference Between dmesg and journalctl

| dmesg | journalctl |
|---------|---------|
| Kernel Logs Only | All Logs |
| Temporary | Persistent |
| Hardware Focused | System-wide |
